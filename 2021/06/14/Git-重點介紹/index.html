Posted on 2021-06-14 Edited on 2021-06-17

版本管控的種類

  • 集中式版本管控
    • 特定的伺服器才會有完整的版本庫及儲存庫
    • 所有人的版本全部集中放在某台電腦管理
    • 常見的版控系統
      • TFS
  • 分散式版本管控
    • 團隊中每個人都擁有完整的版本庫或儲存庫
    • 常見的版控系統
      • Git
      • Mercurial

刪除分支

不能刪除當下所在分支

  • 強制刪除分支

    git branch -D br1

  • 刪除分支，如果未合併，無法刪除

    git branch -d br1

  • 刪除已經合併的所有分支，只能在MacOS/Linux才可以執行

    git branch –merged | egrep -v “(^*|master|main|dev)” | xargs git branch -d

總共三個步驟

  1. 找出已經合併的分支
  2. 略過* or master or main or dev的分支名稱
  3. 刪除剩下分支

什麼情況會建立分支

  • 多人開發
  • 新功能開發
  • 分支規矩分為
    • git flow
    • github flow

發生衝突

當發生衝突當下，有幾種解決方法

  • 放棄當下的合併，復原到還沒合併的狀態
  • 手動修復，與協作對象一同討論後，合併起來

合併

fast-forward(快轉)

當a要合併b，但是b卻比a還要往前一個版本時，a做merge b會直接將b蓋掉，不會產生分支的線圖就稱為快轉。

git merge b –no-ff

  • 當develop比main分支還要前面時，為了使分支有出去再回來的效果，會關閉快轉機制

  • 當server上有人跟你修改同樣的分支，pull下來並不需要有出去又回來的線圖，就會開啟快轉機制(因為是同樣的分支)

不做快轉，並且先不要合併，會處於合併中的狀態

git merge –no-ff –no-commit develop

壓縮合併，會將develop直接截斷，合併出一個版本

git merge –squash develop

git pull

實際上等同於git fetch + git merge

  • git fetch 是將遠端儲存庫下載下來
